HTML

  • The head element contains information about the webpage or metadata.
  • The body element represents the visible content shown to the user.
  • Elements as visible contents on a webpage (i.e. text, image, header, list)
  • Elements as partitions on a webpage (i.e. meta-related, content division, semantic)
  • Element- holds the content vs Attribute- describes the characteristic in detail

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A < link > element creates the connection to the external file.
  • The rel attribute specifies the relationship between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external resource

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A
  • git commit -m "name/ title"
  • git pull origin main
  • git push origin feature/add-something
  • a feature branch is where we make commits of all the changes
  • a pull request or PR is issued to merge a comparison (feature) branch into the base/ default (main) branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Array is group of related data in a variable
  • A function's name is followed by parentheses () e.g. function helloWorld()
